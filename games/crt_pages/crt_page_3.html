Console: Powered by ChaptGPT

Sadly, this was the only picture of the ChatGPT console working as subsequent updates to OpenAI's API have rendered this version un-usable in 2024.

To re-create the 80s/90s retro computer feel, I added a console powered by ChaptGPT.

This was achieved by using Eddy's ChatGPT plugin that makes API calls to ChatGPT when provided with a token. Pretty straightforward and kinda brutforce. For more info on Eddy's plugin check out his game Yandere AI Girlfriend.

How it works

Short answer is: black magic. I made a custom scriptable that stores pre-written prompts. These information includes:

  • How should the console behave (personality traits)
  • What languages to use
  • What are the concepts*
  • Which concepts can / should the player know about at this stage

A concept is a game-rule / mechanic phrased in in-game terminology. Concepts are stored in Databases which are lists of strings with a name-description pair. The description is used to prompt ChatGPT to make sure it knows what the concepts are and what to say. The idea is that: based on player progress, they will unlock more concepts so that the console can tell them more lore / mechanics.

Purpose?

In my opinion, the best usage for LLMs in games is to make it an interactive in-game wiki. If executed well it can revolutionize tutorialization and fragmented storytelling. It should serve as an addition on top of the core experience instead of putting it front and center as the core experience itself. (sorry Eddy)

BUT! I hope someone proves me wrong. AI games are fun.

At least at the time when I made this, LLMs are simply too difficult to control from a designer's standpoint. (maybe I'm just shit too) It's impossible to predict how will it behave once its in players' hands. That can become very annoying very quickly.
This is also why I opt for the Console to only be an assistant, but not to directly affect gameplay.

Console Commands

It's a console. It has to have some commands in some capacity.

Commands are created making use of UnityEvents. Each command should be a scriptable object but I ran out of time to implement it. However they are still easy to create as arrays each with their designated enum. It's created this way so that I can easily utilize UnityEvents to have a clean and visually clear structure to implement and review all the functions that the command is calling.
